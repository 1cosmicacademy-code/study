# قواعد مراجعة الجودة — Quality Review Rules

## 1. عملية المراجعة

### 1.1 التوقيت
- **قبل البناء**: بعد اكتمال جميع التعديلات
- **بعد البناء**: فور انتهاء `mkdocs build`
- **قبل الإغلاق**: مراجعة نهائية شاملة

### 1.2 الأولويات
- **🔴 الأولوية 1**: كل ما يمنع المستخدم من قراءة المحتوى
- **🟠 الأولوية 2**: كل ما يشتت انتباه المستخدم أو يربكه
- **🟡 الأولوية 3**: تحسينات تجميلية وتنسيقية
- **🔵 الأولوية 4**: اقتراحات للمستقبل

## 2. قائمة المراجعة الإلزامية

### 2.1 هيكل الملف
- [ ] H1 واحد فقط
- [ ] ترقيم الأقسام صحيح
- [ ] روابط العودة موجودة
- [ ] الفاصل `---` بين الأقسام

### 2.2 الجداول
- [ ] جميع الأعمدة مملوءة
- [ ] تنسيق المحاذاة صحيح
- [ ] لا توجد خلايا فارغة غير مقصودة
- [ ] صف الفاصل موجود (|---------|)

### 2.3 الروابط
- [ ] الروابط الداخلية تشير لملفات موجودة
- [ ] روابط يوتيوب صالحة (معرف الفيديو صحيح)
- [ ] روابط العودة تعمل
- [ ] لا توجد روابط مقطوعة

### 2.4 المحتوى
- [ ] لا أخطاء إملائية واضحة
- [ ] النطق الصوتي دقيق
- [ ] الأمثلة مفهومة ومناسبة للمستوى
- [ ] المحتوى مناسب للمستوى المعلن (A1/A2)

## 3. التحقق من البناء (MkDocs)

### 3.1 الأوامر
```bash
# بناء الموقع
mkdocs build

# بناء مع تفاصيل الأخطاء
mkdocs build --verbose

# تشغيل الخادم المحلي للاختبار
mkdocs serve
```

### 3.2 ما يتم فحصه
- عدم وجود `ERROR` في المخرجات
- مراجعة `WARNING` وتحديد مدى خطورتها
- التأكد من ظهور جميع الصفحات في `site/`
- التحقق من أن ملفات `.en.md` تولد صفحات إنجليزية

### 3.3 الأخطاء الشائعة
- ملفات مفقودة غير مسجلة في `mkdocs.yml`
- أخطاء YAML في front matter
- مسارات غير صحيحة في الروابط
- ترميز UTF-8 غير صحيح

## 4. التوثيق

### 4.1 تقرير الجودة
كل مراجعة تُنتج تقريراً موقعاً بالتاريخ، يحتوي:
- حالة البناء (نجاح/فشل/تحذيرات)
- قائمة الملفات التي تمت مراجعتها
- الأخطاء مصنفة حسب الأولوية
- التوصيات والخطوات التالية

### 4.2 تتبع الأخطاء
- الأخطاء المتكررة تُسجل في ملف منفصل
- كل خطأ له: وصف، ملف، خطورة، حالة
- ترحل الأخطاء غير المحلولة إلى الجلسة التالية

## 5. الأدوات المسموحة
- العمليات المسموحة: Read, Glob, Grep, Bash (لـ mkdocs build فقط)
- ممنوع: التعديل المباشر على المحتوى دون الرجوع للفريق
- مسموح: الإبلاغ عن الأخطاء فقط
- Bash مسموح فقط لأوامر mkdocs و git status
