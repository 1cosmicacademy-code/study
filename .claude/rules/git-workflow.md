# سير عمل Git

## المبادئ
- لا تعمل على main — أنشئ فرعاً جديداً لكل مهمة
- اسم الفرع: `type/description` (مثلاً `feat/add-auth`, `fix/login-bug`)
- رسالة commit واضحة بالعربية أو الإنجليزية

## التعليمات
- استخدم `git status` قبل أي commit لتتأكد من الملفات
- commit بعد كل مهمة مكتملة (لا تنتظر تراكم التغييرات)
- اسأل قبل عمل push أو PR
- نهاية commit:
  ```
  Co-Authored-By: Claude <noreply@anthropic.com>
  ```
