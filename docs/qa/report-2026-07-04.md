## 4. قائمة المشاكل الموحدة

### 🔴 ID-01: جميع صفحات EN تستخدم dir="rtl" بالخطأ
| الحقل | القيمة |
|-------|--------|
| **النوع** | UI / Accessibility — حرج |
| **الصفحات** | جميع صفحات `/en/` (~80 صفحة) |
| **الوصف** | كل صفحات الإنجليزية تحمل `<body dir="rtl">` رغم أن `lang="en"` صحيح. اتجاه الصفحة يجب أن يكون `ltr` للإنجليزية. هذا يؤثر على محاذاة النص، قراءة القوائم، وترقيم الصفحات. |
| **السبب الجذري** | السطر `direction: rtl` في `mkdocs.yml` (سطر 9) يُطبق على كل الصفحات عبر القالب |
| **خطوات إعادة الإنتاج** | 1. افتح http://localhost:8000/en/ — 2. افحص `<body dir="rtl">` — 3. لاحظ أن المحتوى بمحاذاة يمنى |
| **التأثير** | 100% من المستخدمين الإنجليز يتلقون تجربة معكوسة — المشكلة الأكثر تأثيراً في الموقع |
| **التوصية** | إزالة `direction: rtl` العام من mkdocs.yml. إضافة سكريبت يضبط `dir` ديناميكياً حسب `lang`، أو استخدام قوالب منفصلة لكل لغة. |
| **الملف المقترح للتعديل** | `mkdocs.yml` — إعداد `direction` |
