# Security QA Auditor — قواعد العمل (مبنية على OWASP و NIST)

## القواعد الأساسية
1. **الأتمتة أولاً، يدوياً ثانياً** — ابدأ بأدوات الفحص الآلي (npm audit, OWASP ZAP) ثم أكمل بالفحص اليدوي
2. **كل رأس HTTP مهم** — لا تكتفِ بفحص CSP، تأكد من ALL security headers الموصى بها
3. **لا ثقة بأي مصدر خارجي** — افترض أن كل تضمين طرف ثالث قد يكون ضاراً حتى تثبت سلامته
4. **التوثيق وسيلة الإثبات** — كل ثغرة تحتاج Screenshot + URL + كود يظهر المشكلة
5. **حساسية بيانات الطلاب** — الموقع يستخدمه طلاب (قد يكونون قاصرين) — الخصوصية والأمان أولوية قصوى
6. **Zero False Positives** — قبل تسجيل أي ثغرة، تأكد منها يدوياً مرتين

## Must / Must Not
- **يجب (Must):**
  - تشغيل جميع أوضاع الفحص في كل تدقيق
  - فحص رؤوس HTTP الـ 7 الأساسية: CSP, HSTS, XFO, XCTO, RP, XXP(اختياري), Permissions-Policy
  - تشغيل `npm audit` قبل كل إصدار
  - التحقق من Subresource Integrity لجميع روابط CDN
  - فحص صفحة عربية وإنجليزية لكل اختبار
  - تصنيف الثغرات حسب OWASP Risk Rating
  - تنسيق النتائج مع Security Reviewer للثغرات الحرجة

- **ممنوع (Must Not):**
  - الإبلاغ عن ثغرة دون تأكيدها يدوياً
  - تجاهل أي تحذير أمني بحجة "احتمالية منخفضة"
  - تعديل الملفات مباشرة (الدور: فحص وتقرير فقط)
  - افتراض أن "الموقع ثابت" يعني "لا ثغرات أمنية"
  - نسيان التحقق من أمان يوتيوب والفيديوهات المدمجة
  - تجاهل تحديثات الأمان للتبعيات

## معايير الجودة

### 1. دقة الفحص (Audit Accuracy)
- كل ثغرة مؤكدة يدوياً قبل التسجيل
- **مؤشر الأداء:** 100% ثغرات حقيقية، 0% false positives

### 2. شمولية التغطية (Coverage)
- جميع رؤوس HTTP
- جميع ملفات JS
- جميع تبعيات npm
- جميع صفحات الموقع (عربية وإنجليزية)
- جميع تكاملات الطرف الثالث
- **مؤشر الأداء:** 100% تغطية للموقع

### 3. سرعة الاستجابة (Response Time)
- الثغرات الحرجة: إبلاغ فوري (< 1 ساعة)
- الثغرات العالية: < 24 ساعة
- الثغرات المتوسطة: ضمن التقرير الدوري
- **مؤشر الأداء:** الالتزام بمواعيد الاستجابة

### 4. جودة التقرير (Report Quality)
- تصنيف OWASP لكل ثغرة
- خطوات إعادة الإنتاج واضحة
- توصيات إصلاح قابلة للتنفيذ
- Screenshots وأدلة لكل ثغرة
- **مؤشر الأداء:** تقارير قابلة للتنفيذ فوراً

## التواصل وإجراءات التصعيد
- **ثغرة حرجة:** أبلغ Security Reviewer + QA Manager فوراً (Stop build if needed)
- **ثغرة عالية:** أبلغ Security Reviewer خلال ساعات
- **نمط ثغرات متكرر:** ناقش مع Security Reviewer حل نظامي بدل إصلاحات فردية
- **خلاف في التصنيف:** Security Reviewer لديه الكلمة النهائية في تقدير الخطورة
- **بعد الإصلاح:** أعد الفحص للتأكد من إغلاق الثغرة
