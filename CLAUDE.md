# مشروع دراسة — تعليمات Claude Code

## عن المشروع
- المشروع: **دراسة** — مجال العمل يُحدد لاحقاً
- اللغة الأساسية: العربية (للشرح والتوجيه)
- الكود (إن وجد): حسب ما يتطلبه المشروع

## سير العمل
1. **Explore → Plan → Implement → Commit**
2. استخدم Plan mode للمهام غير الواضحة أو متعددة الملفات
3. المهام البسيطة (تعديل سطر، تصحيح خطأ مطبعي) — نفذ مباشرة
4. استخدم `/clear` بين المهام المختلفة لتفريغ السياق
5. سمِّ الجلسات بـ `/rename` ليسهل الرجوع إليها

## التفويض والمراجعة
1. أطلب منك مهمة → أوكّلها لـ subagent → ترجع بنتيجة
2. أنت تراجع النتيجة وتقرر الخطوة التالية
3. للتحقيقات المعقدة — استخدم subagent بدل قراءة كل ملف بنفسي
4. للتدقيق — subagent يراجع الكود بعيون جديدة

## جودة العمل
- **xhigh effort** — مستوى التفصيل الأقصى (المفضل)
- قدّم وسيلة تحقق مع كل مهمة: اختبار، build، screenshot
- اعرض الأدلة، لا تكتفِ بالتأكيد اللفظي
- راجع النتائج قبل الاعتماد عليها — أحياناً تكون "plausible but wrong"

## أكواد Bash والأوامر
- استخدم CLI tools: gh، npm/yarn/pnpm، git
- فضّل الأداة المناسبة على Bash (Read, Grep, Glob, Edit, Write)

## الذاكرة والتذكر
- Auto memory مفعّل — كلود يكتب ملاحظاته تلقائياً
- اقرأ `MEMORY.md` عند بداية كل جلسة
- أضف للذاكرة عند تكرار نفس التصحيح أو المعلومة

## مشروع "دراسة" — تعليمات خاصة
- `h:\دراسة` هو المجلد الرئيسي
- `/docs` — للمستندات والمراجع
- `.claude/` — إعدادات Claude Code
